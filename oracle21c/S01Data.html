데이터의 종류와 저장법

데이터(Data)

  • 정의: 사실이나 관찰, 측정 등을 통해 얻은 값 (숫자, 문자, 기호 등)
  • 특징: 가공되지 않은 원재료 성격, 단순히 모아두면 의미 제한적
  • 정보와의 차이: 데이터를 가공·분석·해석하면 정보(Information)가 됨
  • 예시:
    • 데이터: 100, 95, 80, 70 (시험 점수)
    • 정보: 이 반의 평균 점수는 86점

데이터베이스(Database)

  • 정의: 다수의 사용자가 공유할 수 있도록 체계적으로 저장·관리되는 데이터의 집합
  • 특징
    • 통합성: 여러 응용에서 사용 가능
    • 저장성: 대량의 데이터 구조적 저장
    • 공유성: 여러 사용자와 프로그램이 동시에 접근 가능
    • 독립성: 프로그램 변경에도 데이터 영향 최소
  • 구성 요소
    • 데이터(Data): 실제 값
    • 스키마(Schema): 데이터 구조와 제약 조건
    • DBMS(Database Management System): 관리 소프트웨어
    • 사용자(User): 데이터를 조작하는 주체
  • DBMS의 역할
    • 데이터 정의: 생성, 구조 지정
    • 데이터 조작: 추가, 조회, 수정, 삭제 (CRUD)
    • 데이터 제어: 보안, 권한, 동시성 제어, 복구
  • 대표 DBMS: Oracle, MySQL, PostgreSQL, SQL Server

관계형 데이터(Relational Data)

  • 정의: 행(Row)과 열(Column) 구조의 테이블 형태로 저장
  • 특징
    • 정형 데이터 관리에 적합
    • 각 테이블은 기본키(Primary Key)로 행 식별
    • 테이블 간 관계는 외래키(Foreign Key)로 연결
    • SQL(Structured Query Language) 사용
  • 장점: 데이터 일관성, 무결성, 보안 관리 용이 / 복잡한 질의 처리에 강점
  • 예시 테이블
    • 학생(Student): StudentID, Name, Age, Major
    • 수강(Course): CourseID, Title, Credits
  • 대표 DBMS: Oracle, MySQL, PostgreSQL

비관계형 데이터(Non-Relational Data, NoSQL)

  • 정의: 테이블 구조를 따르지 않고 다양한 형태의 데이터 저장
  • 특징
    • 비정형·반정형 데이터(JSON, 문서, 그래프 등) 저장 가능
    • 스키마가 고정되지 않아 구조 변경 자유로움
    • 대용량 처리 및 확장성 우수
    • SQL 대신 각 DB 고유 방식 사용
  • 종류와 예시
    • 문서(Document) 기반: JSON, BSON / MongoDB
    • 키-값(Key-Value) 기반: Redis, DynamoDB
    • 그래프(Graph) 기반: Neo4j
    • 컬럼(Column) 기반: Cassandra, HBase
  • 장점: 유연한 구조, 실시간 처리, 분산 시스템 확장 용이

비교 요약

  • 관계형: 정형 데이터, 고정된 스키마, SQL 사용, 복잡한 질의 처리 강점
  • 비관계형: 비정형·반정형 데이터, 유연한 구조, 확장성·속도 강점
