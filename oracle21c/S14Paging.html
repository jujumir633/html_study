오라클의 페이징

페이징(Paging) 개념

  • 데이터를 한 번에 전부 가져오지 않고 일정 단위(페이지)로 나누어 조회하는 방식
  • 예: 게시판에서 1페이지당 5건씩 출력
  • 핵심: "몇 번째 행부터 몇 개를 가져올지"를 결정 (Offset / Limit)

1. ROWNUM 이용 (전통 방식)

  • 오라클에서 모든 버전에서 사용 가능
  • ROWNUM은 결과 집합에 순서대로 붙는 임시 행 번호
  • 정렬이 있을 경우, 반드시 인라인 뷰로 정렬을 먼저 확정해야 함

정렬 없는 경우

-- 페이지당 5건, 2페이지(=6~10행) 조회
SELECT *
FROM (
  SELECT ROWNUM AS rnum, e.*
  FROM emp e
  WHERE ROWNUM <= 10
)
WHERE rnum BETWEEN 6 AND 10;

정렬 있는 경우

-- 급여 내림차순 정렬 후 2페이지(=6~10행) 조회
SELECT *
FROM (
  SELECT ROWNUM AS rnum, x.*
  FROM (
    SELECT e.*
    FROM emp e
    ORDER BY sal DESC    -- ① 정렬 확정
  ) x
  WHERE ROWNUM <= 10     -- ② 상한 적용
)
WHERE rnum BETWEEN 6 AND 10; -- ③ 원하는 구간 추출

2. ROW_NUMBER() 분석 함수 이용

  • 오라클 9i 이상부터 지원
  • ROW_NUMBER() 함수는 지정한 정렬 기준에 따라 순번을 부여
  • 정렬이 필요한 경우 ROWNUM보다 직관적이고 실무에서 가장 많이 사용됨
-- 급여 높은 순으로 2페이지(=6~10행) 조회
SELECT *
FROM (
  SELECT e.*, ROW_NUMBER() OVER (ORDER BY sal DESC) AS rnum
  FROM emp e
)
WHERE rnum BETWEEN 6 AND 10;

3. OFFSET / FETCH 구문 (Oracle 12c 이상)

  • SQL 표준 방식으로, 가장 직관적이고 간단한 페이징 방법
  • OFFSET n ROWS : 앞의 n개 행을 건너뜀
  • FETCH NEXT m ROWS ONLY : 그 다음 m개 행을 가져옴
-- 급여 높은 순으로 2페이지(=6~10행) 조회
SELECT ename, sal
FROM emp
ORDER BY sal DESC
OFFSET 5 ROWS FETCH NEXT 5 ROWS ONLY;

방식 비교

  • ROWNUM: 모든 버전에서 사용 가능, ORDER BY 시 주의 필요
  • ROW_NUMBER(): 9i 이상, 정렬 기준 명확, 실무 표준
  • OFFSET / FETCH: 12c 이상, SQL 표준, 가장 직관적

MySQL 페이징 기본

  • MySQL에서는 LIMIT으로 페이징을 처리한다
  • 형식: LIMIT 시작위치, 개수 (시작은 0부터)
  • LIMIT 개수 OFFSET 시작위치 형태도 가능하다

예시

-- 1페이지 (1~5행)
SELECT * FROM emp
ORDER BY sal DESC
LIMIT 0, 5;

-- 2페이지 (6~10행)
SELECT * FROM emp
ORDER BY sal DESC
LIMIT 5, 5;

-- 3페이지 (11~15행)
SELECT * FROM emp
ORDER BY sal DESC
LIMIT 10, 5;

-- OFFSET 문법
SELECT * FROM emp
ORDER BY sal DESC
LIMIT 5 OFFSET 5;

pay_history 페이징 연습문제

테이블 구조: PAY_ID, EMPNO, PAY_DATE, AMOUNT, BONUS

① ROWNUM 방식

  1. 정렬 없이 1페이지(1~5행)를 조회하라. (페이지당 5건) 힌트: 안쪽에서 ROWNUM <= 5, 바깥에서 rnum BETWEEN 1 AND 5
  2. 정렬 없이 2페이지(6~10행)를 조회하라. (페이지당 5건)
  3. EMPNO=1001 사원의 데이터만 대상으로 1페이지(1~5행)를 조회하라.

② ROW_NUMBER() 함수 방식

  1. 급여일(PAY_DATE) 내림차순으로 정렬 후, 1페이지(1~5행)를 조회하라. 힌트: ROW_NUMBER() OVER (ORDER BY PAY_DATE DESC)
  2. 금액(AMOUNT) 높은 순으로 2페이지(6~10행)를 조회하라.
  3. 보너스(BONUS)가 있는 건만 대상으로 PAY_DATE DESC 정렬 후 1페이지(1~5행)를 조회하라.

③ OFFSET / FETCH 방식 (Oracle 12c+ 또는 MySQL)

  1. PAY_DATE DESC 기준으로 1페이지(1~5행)를 조회하라. 힌트: OFFSET 0 ROWS FETCH NEXT 5 ROWS ONLY
  2. AMOUNT DESC 기준으로 2페이지(6~10행)를 조회하라.
  3. EMPNO=2000 사원의 데이터를 PAY_DATE DESC 정렬 후 3페이지(11~15행)를 조회하라.
  4. 최근 30일 이내 데이터만 대상으로 PAY_DATE DESC 정렬 후 1페이지(1~5행)를 조회하라.
