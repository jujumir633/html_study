DML 데이터 조작어

DML (Data Manipulation Language) 개요

  • DML은 데이터베이스의 데이터를 다루는 언어(CRUD).
  • CRUD: Create(등록)=INSERT, Read(조회)=SELECT, Update(수정)=UPDATE, Delete(삭제)=DELETE

등록 (INSERT)

단일 행

INSERT INTO member (id, name, email)
VALUES (1, '홍길동', 'hong@test.com');

여러 행

  • INSERT ALL
INSERT ALL
  INTO member (id, name, email) VALUES (2, '김철수', 'kim@test.com')
  INTO member (id, name, email) VALUES (3, '이영희', 'lee@test.com')
SELECT 1 FROM dual;
  • INSERT INTO ... SELECT (다른 테이블에서 복사)
INSERT INTO member (id, name, email)
SELECT id, name, email
FROM backup_member;

참고: MySQL의 VALUES (...), (...) 다중 삽입 구문은 오라클에서 직접 지원하지 않음.

조회 (SELECT)

기본

SELECT id, name
FROM member;

전체 컬럼

SELECT *
FROM member;

WHERE (동등 비교)

SELECT id, name
FROM member
WHERE name = '홍길동';

수정 (UPDATE)

UPDATE member
SET email = 'new@test.com', name = '홍길동2'
WHERE id = 1;

삭제 (DELETE)

DELETE FROM member
WHERE id = 1;

트랜잭션

  • COMMIT: 작업 확정
  • ROLLBACK: 작업 취소
  • SAVEPOINT: 중간 저장점
SAVEPOINT sp1;
UPDATE member SET grade = 'VIP' WHERE id = 2;
ROLLBACK TO sp1;  -- 부분 취소
COMMIT;           -- 최종 확정

예제 테이블 (선택)

CREATE TABLE member (
  id        NUMBER PRIMARY KEY,
  name      VARCHAR2(50) NOT NULL,
  email     VARCHAR2(100),
  grade     VARCHAR2(10) DEFAULT '일반',
  reg_date  DATE DEFAULT SYSDATE
);

CREATE SEQUENCE seq_member START WITH 1 INCREMENT BY 1;
