관계형 데이터와 저장소

관계형 데이터베이스(RDB)의 특징

  • 데이터 저장: 미리 정의된 스키마(구조)에 따라 데이터를 테이블로 저장
  • 무결성 보장: 데이터의 정확성과 일관성을 유지
  • 트랜잭션 지원: ACID(원자성, 일관성, 고립성, 지속성) 보장
  • SQL 사용: 데이터 정의/조회/수정/삭제를 모두 SQL로 처리

장점

  • 데이터의 정확성(Integrity)과 일관성(Consistency) 유지
  • 표준 SQL 사용 → 다양한 시스템/응용 프로그램과 호환 가능
  • 강력한 트랜잭션 기능 지원 → 은행, 회계, 쇼핑몰 등 안전성이 중요한 곳에 적합

단점

  • 스키마(구조) 변경이 까다로워 유연성이 낮다
  • 대규모 데이터에 대한 수평 확장(Scale-out)이 어렵다
  • 비정형 데이터(이미지, 로그, JSON 등) 처리에는 적합하지 않다

대표 제품

  • Oracle Database
  • MySQL
  • PostgreSQL
  • Microsoft SQL Server

활용 예시

  • 은행 계좌 관리 시스템 → 거래 기록, 잔액 관리
  • 쇼핑몰 주문 관리 → 고객, 주문, 상품 테이블
  • 학교 학생 관리 → 학생, 수업, 성적 테이블

기본 구성 요소

  • 테이블(Table): 데이터 저장 단위
  • 행(Row/Record): 데이터 한 건
  • 열(Column/Attribute): 속성(필드)
  • 기본 키(Primary Key): 행을 유일하게 식별
  • 외래 키(Foreign Key): 다른 테이블의 기본 키를 참조해 관계 표현

트랜잭션과 ACID

  • 트랜잭션(Transaction): 한 번에 처리되어야 하는 작업 단위
    예) A계좌 10,000원 출금 → B계좌 10,000원 입금
  • ACID 네 가지 특성
    • 원자성(Atomicity): 전부 수행되거나 전혀 수행되지 않음
    • 일관성(Consistency): 전후로 제약/규칙을 항상 만족
    • 고립성(Isolation): 동시에 실행돼도 서로 간섭 없음
    • 지속성(Durability): 완료 결과는 영구 저장(장애 후에도 유지)

테이블 예시

  • Student(학생) 테이블과 Course(수강) 테이블의 1:N 관계
  • Student.학번 = Course.학번(외래키)로 연결

Student 테이블

학번 (PK) 이름 나이 학과
1001 홍길동 20 컴퓨터공학
1003 홍길동 20 컴퓨터공학
1002 김철수 22 경영학

Course 테이블

수강번호 (PK) 학번 (FK → Student.학번) 과목명
C01 1001 데이터베이스
C02 1002 마케팅
C03 1001 운영체제

간단 SQL 예시

  • 테이블 생성, 기본키/외래키 지정
-- Student 테이블
CREATE TABLE Student (
  student_id NUMBER PRIMARY KEY,
  name       VARCHAR2(50),
  age        NUMBER,
  major      VARCHAR2(50)
);

-- Course 테이블 (Student.student_id 외래키 참조)
CREATE TABLE Course (
  course_id  VARCHAR2(10) PRIMARY KEY,
  student_id NUMBER REFERENCES Student(student_id),
  title      VARCHAR2(50)
);

-- 샘플 데이터
INSERT INTO Student VALUES (1001, '홍길동', 20, '컴퓨터공학');
INSERT INTO Student VALUES (1002, '김철수', 22, '경영학');

INSERT INTO Course VALUES ('C01', 1001, '데이터베이스');
INSERT INTO Course VALUES ('C02', 1002, '마케팅');
INSERT INTO Course VALUES ('C03', 1001, '운영체제');

-- 조인 조회(학생과 수강 과목)
SELECT s.student_id, s.name, c.course_id, c.title
FROM Student s
JOIN Course  c ON c.student_id = s.student_id
ORDER BY s.student_id, c.course_id;
