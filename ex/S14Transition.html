1) 트랜지션이란?

  • 스타일 값이 바뀔 때, 변화가 즉시 “뚝” 바뀌지 않고 지정한 시간과 속도 곡선에 따라 부드럽게 변하도록 만드는 CSS 기능
  • 상태 전환(예: :hover, :focus, 클래스 토글 등) 사이의 “중간값”을 브라우저가 자동 계산하여 프레임 단위로 그려줌
  • JavaScript가 없어도 동작하며, 시작·끝 상태만 선언하면 됨
  • 수치가 없는 속성은 트랜지션을 적용할 수 없다.(display:none,hidden)

2) 애니메이션이 동작하는 방식(개념)

  • 시작 상태와 끝 상태가 정해지면, 브라우저가 일정한 프레임(보통 60fps 목표) 간격으로 중간 값을 보간
  • duration(지속시간)과 timing-function(속도 곡선)에 따라 각 프레임의 변화 폭이 달라짐
  • transition은 “상태 변화가 발생했을 때만” 재생되며, 명시적 키프레임 없이도 부드러운 변화를 제공

3) 브라우저는 애니메이션을 어떻게 그리나(렌더링 관점)

  • 레이아웃(Layout): 크기·위치가 변하는 속성(top/left/width 등)을 바꾸면 레이아웃 재계산을 수행
  • 페인트(Paint): 색, 배경, 테두리 등은 다시 칠하기가 필요
  • 합성(Compositing): transform, opacity 등은 보통 합성 단계에서 갱신 가능하여 성능 유리
  • 실무 팁: 이동은 left/top 대신 transform: translate()를, 등장/퇴장은 opacity와 transform을 우선 고려

4) 트랜지션의 구성 요소(속성)

  • transition-property: 어떤 속성을 부드럽게 바꿀지 지정 (예: background-color, transform). 기본값은 all
  • transition-duration: 변화가 완료되는 시간 (예: 300ms, 0.3s)
  • transition-timing-function: 속도 곡선(ease, linear, ease-in/out, cubic-bezier())
  • transition-delay: 시작 전 대기 시간 (예: 0.2s)
  • 단축형: transition: property duration timing-function delay; 순서로 작성

5) 속도(타이밍) 곡선 이해

  • linear: 처음부터 끝까지 같은 속도
  • ease: 초반 빠르고 후반 느림(기본값)
  • ease-in / ease-out / ease-in-out: 가속·감속 패턴을 다르게 적용
  • cubic-bezier(x1, y1, x2, y2): 사용자 정의 곡선으로 세밀한 제어
    • cubic-bezier 시각 편집기:cubic-bezier.com
    • 여러 이징 모음/미리보기:easings.net
    • 프리셋 애니메이션/코드 생성:animista.net

7) 주의 사항과 베스트 프랙티스

  • 가능하면 합성 단계에서 끝나는 속성(transform, opacity)을 애니메이션
  • 너무 많은 요소에 긴 duration/무거운 속성 애니메이션을 동시에 주지 않기
  • 상태 전환이 잦다면 duration을 짧게(지루함/딜레이 방지)

8) 실습 과제(짧게)

  • 1: 버튼 색상과 border-radius가 0.2초 동안 바뀌도록 트랜지션 설정
  • 2: 카드에 hover 시 transform: translateY(-8px), box-shadow 변경(0.3초 ease-out)
  • 3: 메뉴 항목에 hover 지연 0.1초와 ease-in 적용, focus 시에도 동일 동작
  • 4: cubic-bezier로 “빠르게 시작-매우 부드럽게 멈춤” 곡선을 만들고 적용
