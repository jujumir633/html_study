비동기 함수와 이벤트루프

비동기 함수란?

  • 프로그램이 특정 작업을 기다리지 않고, 다음 작업을 계속 실행할 수 있도록 하는 함수
  • 파일 읽기, 네트워크 요청, 타이머 같은 시간이 오래 걸리는 작업에서 활용
  • 비동기를 사용하면 "응답 없는 상태"를 줄이고, 사용자 경험을 개선

동기와 비동기

  • 동기(Synchronous): 한 작업이 끝나야 다음 작업이 실행됨. (예: 라면 물이 끓을 때까지 아무것도 못 함)
  • 비동기(Asynchronous): 기다리지 않고 다른 작업을 먼저 진행하다가, 결과가 준비되면 처리. (예: 물 끓는 동안 반찬 준비)

비동기의 필요성

  • 자바스크립트는 싱글 스레드로 동작하므로, 긴 작업이 동기 방식으로 실행되면 브라우저가 멈춘 것처럼 보인다.
  • 예: 서버에서 데이터를 가져올 때까지 화면이 멈추면 사용자 경험(UX)이 크게 나빠진다.
  • 비동기를 사용하면 긴 작업을 따로 맡겨두고, 메인 코드는 계속 실행되어 UI가 부드럽게 동작한다.
  • 즉, 비동기는 브라우저가 끊기지 않고 반응성 있는 동작을 하기 위해 꼭 필요하다.

멀티스레드 환경의 비동기

  • 일반적인 언어(Java, C# 등)는 여러 스레드를 동시에 실행하여 비동기를 처리한다.
  • 예: 다운로드는 별도 스레드에서 실행하고, 메인 스레드는 UI 반응을 유지한다.
  • 즉, 병렬(Parallel)로 여러 작업이 동시에 돌아가는 방식.

자바스크립트의 비동기 처리

  • 자바스크립트는 싱글 스레드 언어라서 한 번에 한 줄만 실행한다.
  • 하지만 이벤트 루프(Event Loop) 구조를 통해 비동기 작업을 지원한다.
  • 멀티스레드 대신 이벤트 루프와 태스크 큐를 이용해 동시성을 구현한다.

이벤트 루프(Event Loop)의 동작 원리

  • Call Stack: 동기 코드가 실행되는 공간.
  • Web APIs: 타이머, fetch, XMLHttpRequest 등 비동기 작업을 브라우저에 맡김.
  • Task Queue: 비동기 작업이 끝나면 콜백이 들어와 대기.
  • Event Loop: Call Stack이 비면 Task Queue에서 콜백을 꺼내 실행.

이벤트 루프와 비동기

  • JavaScript는 싱글 스레드 언어 → 한 번에 한 작업만 처리 가능
  • 비동기 작업은 이벤트 루프(Event Loop)와 태스크 큐(Task Queue)를 통해 처리
  • 작업 순서: 호출 스택(Call Stack) → 비동기 작업(Web APIs) → 태스크 큐 → 이벤트 루프 → 다시 호출 스택

자주 쓰이는 비동기 함수

  • setTimeout(callback, ms): 일정 시간이 지난 뒤 콜백을 한 번만 실행.
  • setInterval(callback, ms): 일정 시간 간격마다 콜백을 반복 실행.
  • XMLHttpRequest: 오래된 방식의 비동기 통신. 콜백 기반으로 서버와 데이터 송수신.
  • fetch(url, options): 최신 표준 통신 방식. Promise 기반으로 서버와 데이터 송수신.
