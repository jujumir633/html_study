자바스크립트의 연산자

숫자와 연산 정리

산술 연산자

  • +, -, *, /, %, **(거듭제곱, Math.pow())
  • 정수와 실수 구분 없이 모두 Number 타입으로 연산
  • 예: 11 / 22 → 0.5 (자바는 0, JS는 부동소수점)
  • 예: 2 ** 3 → 8

증감 연산자

  • ++ (증가), -- (감소)
  • 전위(++a): 값을 먼저 증가시키고 반환
  • 후위(a++): 값을 반환한 뒤 증가
  • 후위 연산은 "연산 순서를 양보"한다고 이해
  • 복합 연산 시 기대값과 실제값의 차이가 발생할 수 있음

대입 연산자

  • =, +=, -=, *=, /=, %=, **=
  • a += 100 → a = a + 100
  • a %= 2 → 0이면 짝수, 1이면 홀수 판별에 활용 가능

특수한 상황 (Infinity, NaN)

  • 0으로 나누면 Infinity (자바는 오류 발생)
  • 양수/작은 수로 나눔: 무한대 출력
  • NaN (Not a Number): 수가 아닌데 수학 연산을 시도했을 때 발생
  • NaN도 typeof로 확인하면 number 타입

문자열과 연산

  • 문자열 + 문자열 → 연결(concat)
  • 문자열 + 숫자/불리언 → 모두 문자열로 변환 후 연결
  • 문자열 + null/undefined → 문자열화되어 연결
  • 문자열과 + 를 제외한 *, /, %, ** → 문자열을 숫자로 변환 시도
  • 변환 불가능하면 NaN 발생

형 변환

  • Number("11") → 11 (문자열을 숫자로 변환)
  • Number("안녕") → NaN
  • "11" * 2 → 22 (자동 변환)
  • "11" + 2 → "112" (+는 문자열 우선)
  • *, /, %, ** 연산자는 문자열을 숫자로 변환하려 시도

자바와 자바스크립트 비교

  • 자바: 엄격하고 규칙이 단단한 언어 (정적 타입, 컴파일 시 에러)
  • 자바스크립트: 유연하고 동적 타입 (개발 속도 빠르지만 런타임 오류 발생 가능)

문제

  • 다음 코드의 출력 결과를 쓰시오. 
    let a = 11;
let b = 22;
console.log(a / b);
  • 다음 코드의 출력 결과를 쓰시오. 
    console.log(2 ** 3);
console.log(Math.pow(2, 3));
  • 다음 코드의 출력 결과를 쓰시오. 
    let a = 5;
console.log(++a);
console.log(a++);
console.log(a);
  • 다음 코드의 출력 결과를 쓰시오. 
    let a = 11;
let b = 22;
let sum = (a--) + (b--);
console.log(sum);
console.log(a, b);
  • 다음 코드의 출력 결과를 쓰시오. 
    let a = 10;
a += 5;
a %= 2;
console.log(a);
  • 다음 코드의 출력 결과를 쓰시오. 
    console.log(1 / 0);
console.log(-1 / 0);
  • 다음 코드의 출력 결과를 쓰시오. 
    let str = "안녕";
console.log(str + 10);
console.log(10 + str);
  • 다음 코드의 출력 결과를 쓰시오. 
    let str = "hello";
console.log(str * 2);
console.log(str / 2);
  • 다음 코드의 출력 결과를 쓰시오. 
    console.log("11" * 2);
console.log("11" + 2);
  • 다음 코드의 출력 결과를 쓰시오. 
    console.log(typeof 11.1);
console.log(typeof NaN);
  • let a = 11;
let b = 22;
console.log(a / b);
    정답: 0.5 해설: JS는 정수/실수 구분 없이 모두 부동소수점으로 계산한다. 
  • console.log(2 ** 3);
console.log(Math.pow(2, 3));
    정답: 8, 8 해설: ** 연산자와 Math.pow()는 같은 거듭제곱 연산. 
  • let a = 5;
console.log(++a);
console.log(a++);
console.log(a);
    정답: 6, 6, 7 해설: 전위 증가는 먼저 증가, 후위 증가는 출력 후 증가. 
  • let a = 11;
let b = 22;
let sum = (a--) + (b--);
console.log(sum);
console.log(a, b);
    정답: 33, 10 21 해설: 후위 감소이므로 연산에 11과 22가 사용되고, 이후 각각 10, 21로 감소. 
  • let a = 10;
a += 5;
a %= 2;
console.log(a);
    정답: 1 해설: 10+5=15, 15%2=1. 
  • console.log(1 / 0);
console.log(-1 / 0);
    정답: Infinity, -Infinity 해설: JS는 0으로 나누면 오류 대신 무한대 출력. 
  • let str = "안녕";
console.log(str + 10);
console.log(10 + str);
    정답: "안녕10", "10안녕" 해설: 문자열 + 숫자 = 문자열 연결. 
  • let str = "hello";
console.log(str * 2);
console.log(str / 2);
    정답: NaN, NaN 해설: 문자열 "hello"는 숫자로 변환 불가. 
  • console.log("11" * 2);
console.log("11" + 2);
    정답: 22, "112" 해설: * 연산은 문자열을 숫자로 변환, + 연산은 문자열 연결. 
  • console.log(typeof 11.1);
console.log(typeof NaN);
    정답: "number", "number" 해설: NaN도 숫자 타입으로 취급된다.
