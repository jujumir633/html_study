자바스크립트의 변수와 기본형

1. 자바스크립트의 변수 선언

  • let : 재할당 가능한 변수
  • const : 상수, 재할당 불가
  • var : 옛날 방식, 함수 범위(function scope) → 권장하지 않음
  • 자바스크립트는 **변수 선언 시 타입을 지정하지 않음** → 동적 타입 언어
    • let a = 10;        // 숫자
a = "문자열";      // 자료형 변경 가능

2. 자바스크립트의 기본형 (Primitive Types)

  • Number : 정수/실수를 구분하지 않고 모두 하나의 숫자 타입 
    let x = 10;     // 정수
let y = 3.14;   // 실수
  • String : 문자열, 작은따옴표('), 큰따옴표("), 백틱(`) 모두 사용 가능 
    let s1 = "Hello";
let s2 = 'World';
let s3 = `템플릿 리터럴 ${s1}`;
  • Boolean : 참/거짓 
    let isLogin = true;
  • Undefined : 값이 아직 할당되지 않은 상태 
    let data;
console.log(data); // undefined
  • Null : 의도적으로 "값이 없음"을 지정 
    let value = null;
  • Symbol : 고유하고 유일한 값, 주로 객체 키로 사용 
    let id = Symbol("id");
  • BigInt : 아주 큰 정수 표현 가능 
    let big = 123456789012345678901234567890n;

3. 자바스크립트에서 문자열을 기본형으로 만든 이유

  • 웹에서 문자열 처리가 압도적으로 많이 필요했음 (텍스트, 입력 값, HTML 등)
  • 자바처럼 char + String 두 가지를 두면 복잡해짐
  • 따라서 설계 단계에서 문자 1개도 문자열(String)으로 처리하도록 단순화
  • 내부적으로는 기본형(String) + 임시 객체(Wrapper)를 혼합해 동작
  • 예: 
    let s = "hello";
console.log(s.length);  // 5
// 기본형인데 .length, .toUpperCase() 같은 메서드 사용 가능
// 실행 순간 자바스크립트 엔진이 임시 String 객체로 변환 후 처리

3. typeof 연산자

  • 변수의 자료형 확인 가능
    • typeof 123        // "number"
typeof "hi"       // "string"
typeof true       // "boolean"
typeof undefined  // "undefined"
typeof null       // "object" (버그로 알려진 오래된 특징)
typeof 123n       // "bigint"
