자바스크립트 문자열(String)

배열(Array)이란?

  • 배열은 여러 데이터를 하나의 변수에 순서대로 저장할 수 있는 자료 구조
  • 하나의 이름으로 여러 값을 관리할 수 있음
  • 인덱스(index, 위치 번호)를 사용해 각 요소에 접근 (0부터 시작)
  • 서로 다른 자료형도 한 배열 안에 저장 가능 (자바스크립트의 특징)
  • 반복문과 함께 자주 사용되며, 데이터를 모아 처리할 때 유용

예시

  • let fruits = ["사과", "배", "포도"];
  • console.log(fruits[0]); // "사과"
  • console.log(fruits[2]); // "포도"

문자열의 특징

  • 문자열은 문자로된 배열 ['안','녕','하','세','요','.']로 순서와 길이가 있는 데이터
  • "" ,'',`` 로 문자열 데이터를 생성가능
  • 문자열은 기본형 데이터 타입
  • 문자열은 불변(immutable)하여 한 번 생성되면 수정 불가
  • 문자열 수정 시 항상 새로운 문자열을 반환
  • 메서드를 사용할 때 자동으로 String 객체(자료형)로 변환

문자열이 기본형인 이유

  • 문자열은 숫자, 불린과 함께 가장 기초적인 데이터 타입
  • 가볍게 다루고 자주 쓰기 때문에 값 자체로만 존재해야 함 (복잡한 객체 X)
  • 불변성(immutable)을 유지하기 위해 기본형으로 설계됨 → 한 번 생성된 문자열은 바뀌지 않고, 수정 시 새 문자열이 생성됨
  • 성능상 이점: 문자열을 값 그대로 저장하고 비교하는 것이 객체보다 훨씬 빠름

기본형 문자열인데 메서드를 쓸 수 있는 이유

  • 자바스크립트 엔진이 문자열을 사용할 때 자동으로 String 객체로 래핑해줌
  • 예: "hello".toUpperCase() 실행 시
    • 내부에서 new String("hello") 객체가 잠깐 만들어짐
    • 그 객체의 메서드 toUpperCase() 실행
    • 실행이 끝나면 객체는 사라지고 결과 문자열만 남음
  • 즉, 문자열은 기본형이지만 필요할 때만 객체처럼 동작하도록 만든 것

문자열 작성법과 템플릿 리터럴

  • 작은따옴표('')
    예시: 'Hello'
    예시: '경민이 "Hello"라 말했다'
  • 큰따옴표("")
    예시: "Hello"
    예시: "경민이 'Hello'라 생각했다."
  • 백틱(``) : 템플릿 리터럴
    • 변수를 쉽게 삽입 가능 → `안녕, ${name}`
    • 여러 줄 문자열 작성 가능

    예시: let name = "경민"; console.log(`안녕하세요, ${name}님`); /* 출력: 안녕하세요, 경민님 */

이스케이프 문자

  • 이스케이프 문자란?
    문자열 안에서 특별한 의미를 가지는 문자. \(역슬래시)와 함께 사용.
  • 자주 사용하는 이스케이프 문자들
    • \n : 줄바꿈 (new line)
      예시: "안녕\n하세요"
    • \t : 탭(tab) 간격
      예시: "A\tB"
    • \" : 큰따옴표 출력
      예시: "그는 말했다: \"안녕\""
    • \' : 작은따옴표 출력
      예시: 'It\'s OK'
    • \\ : 역슬래시 출력
      예시: "C:\\Users\\경민"

HTML에서 사용하는 이스케이프 문자(엔티티)

자바스크립트의 \n, \" 같은 이스케이프 문자와는 달리, HTML에서는 <, >, & 등을 직접 쓰면 태그로 인식되기 때문에 엔티티 코드

  • &lt; → 출력: < (less than, 작다 기호)
  • &gt; → 출력: > (greater than, 크다 기호)
  • &amp; → 출력: & (ampersand, 앰퍼샌드)
  • &quot; → 출력: " (큰따옴표)
  • &apos; → 출력: ' (작은따옴표)
  • &nbsp; → 출력:   (공백, non-breaking space)

문자열 간의 비교

  • 자바스크립트 문자열 비교는 **유니코드 코드값(사전식)** 기준으로 비교한다.
  • <, > 연산자를 사용하면 문자열의 각 문자를 왼쪽부터 하나씩 비교한다.
  • 대소문자는 다른 코드값을 가지므로 "A" < "a" 이다.
  • 한글도 유니코드 순서에 따라 비교된다. ("가" < "나")

사용 예시

  • "apple" < "banana" → true (사전식으로 a(97)가 b(98)보다 앞)
  • "a" < "b" → true
  • "Zoo" < "apple" → true ("Z"의 코드값이 "a"보다 작음)
  • "가" < "나" → true (유니코드 순서: 가 → 나 → 다 ...)
  • "안녕" == "안녕" → true (내용이 같으면 true)
  • "사과" == "바나나" → false

문자열과 숫자 비교

숫자로 변환 가능한 문자열

  • 문자열이 숫자 형태라면 자동으로 숫자로 변환되어 비교가 가능하다.
  • 예시:
    10 > "5" // true (문자열 "5" → 숫자 5)
    10 < "11" // true (문자열 "11" → 숫자 11)
    11 == "11" // true (동등 비교에서 "11" → 숫자 11)

숫자로 변환 불가능한 문자열

  • 문자열이 숫자로 바뀔 수 없으면 NaN 이 되어 비교가 불가능하다.
  • NaN과의 모든 비교 결과는 항상 false이다.
  • 예시:
    10 > "안녕" // false ("안녕" → NaN)
    10 < "안녕" // false
    10 == "안녕" // false

문자열의 숫자 변환

  • 문자열을 숫자로 변환하는 방법은 여러 가지가 있다.
  • 문자열이 숫자로 변환 불가능하면 NaN(Not a Number)이 된다.

Number() 함수

  • 문자열 전체를 숫자로 변환
  • Number("123") // 123
  • Number("12.34") // 12.34
  • Number("안녕") // NaN

parseInt()

  • 정수 부분만 변환 (소수점 이하 버림), 뒤쪽 문자열 제거
  • parseInt("123px") // 123 (문자 섞여 있어도 앞부분 숫자는 변환)
  • parseInt("12.34") // 12

parseFloat()

  • 실수 변환 (소수점 유지), 뒤쪽 문자열 제거
  • parseFloat("12.34") // 12.34
  • parseFloat("123px") // 123

단항 연산자 +

  • +수를 양의 수로 변경하는 연산자로 간단하게 문자열을 숫자로 바꾸는 연산자
  • +"123" // 123
  • +"-123" // -123
  • +"12.34" // 12.34
  • +"안녕" // NaN

문자열의 자주 사용하는 함수들

길이와 문자 접근

  • length : 문자열 길이 확인
    예시: "안녕하세요".length // 5
  • charAt(index) : 특정 위치의 문자 반환
    예시: "Java".charAt(0) // "J"
  • [index] : 특정 문자에 접근
    예시: "Java"[1] // "a"

대소문자 변환

  • toUpperCase() : 대문자로 변환
    예시: "hello".toUpperCase() // "HELLO"
  • toLowerCase() : 소문자로 변환
    예시: "WORLD".toLowerCase() // "world"

문자열 검색

  • indexOf("문자") : 위치 반환
    예시: "apple banana".indexOf("banana") // 6
  • includes("문자") : 포함 여부 확인
    예시: "apple".includes("app") // true
  • startsWith("문자") : 시작 여부 확인
    예시: "JavaScript".startsWith("Java") // true
  • endsWith("문자") : 끝나는지 확인
    예시: "JavaScript".endsWith("Script") // true

문자열 추출

  • substring(start, end) : 구간 반환
    예시: "JavaScript".substring(0,4) // "Java"
  • slice(start, end) : 구간 반환 (음수 인덱스 가능)
    예시: "JavaScript".slice(-6) // "Script"
  • substr(start, length) : 특정 길이만큼 반환 (옛 문법)
    예시: "JavaScript".substr(4,6) // "Script"

문자열 분리와 결합

  • split("구분자") : 배열로 분리
    예시: "사과,배,포도".split(",") // ["사과","배","포도"]
  • join("구분자") : 배열을 문자열로 결합
    예시: ["Hello","World"].join(" ") // "Hello World"

문자열 다듬기

  • trim() : 앞뒤 공백 제거
    예시: " hi ".trim() // "hi"
  • trimStart() : 앞 공백 제거
    예시: " hi".trimStart() // "hi"
  • trimEnd() : 뒤 공백 제거
    예시: "hi ".trimEnd() // "hi"

문자열 치환과 반복

  • replace("찾을문자","바꿀문자") : 문자열 치환
    예시: "오늘은 월요일".replace("월요일","화요일") // "오늘은 화요일"
  • repeat(n) : 문자열 반복
    예시: "ha".repeat(3) // "hahaha"

서술형 문제

  • 문자열이 기본형(primitive type)임에도 불구하고 메서드를 사용할 수 있는 이유를 설명하시오.
  • 문자열이 불변(immutable)하다는 의미를 예시와 함께 설명하시오.
  • 자바스크립트에서 문자열 비교가 “사전식 순서”가 아니라 “문자 코드값” 기준이라는 것을 예시로 설명하시오.
  • 문자열을 숫자로 변환하는 방법을 3가지 이상 서술하고, 각각의 차이를 간단히 설명하시오.
  • HTML에서 <, >, & 같은 문자를 문자열로 출력하기 위해 왜 엔티티(escape 문자)를 사용해야 하는지 설명하시오.

문자열 연습 문제 - 정답 및 해설

객관식 / 실습형 문제 정답

  • "JavaScript".length → 10
    문자열의 길이를 반환한다.
  • "Hello".charAt(0) 또는 "Hello"[0] → "H"
    인덱스는 0부터 시작한다.
  • "Hello World".toUpperCase() → "HELLO WORLD"
    모든 문자를 대문자로 변환.
  • "Hello World".includes("World") → true
    포함 여부 확인.
  • "Hello World".startsWith("Hello") → true
    지정한 문자열로 시작하는지 확인.
  • "Hello World".endsWith("ld") → true
    지정한 문자열로 끝나는지 확인.
  • "JavaScript".substring(0,4) → "Java"
    0번부터 3번 인덱스까지 추출.
  • "JavaScript".slice(-6) → "Script"
    뒤에서 6글자를 추출.
  • "사과,배,포도".split(",") → ["사과","배","포도"]
    구분자를 기준으로 문자열을 배열로 변환.
  • ["사과","배","포도"].join("-") → "사과-배-포도"
    배열 요소를 "-"로 연결.
  • " hello ".trim() → "hello"
    앞뒤 공백 제거.
  • "오늘은 월요일입니다.".replace("월요일","화요일") → "오늘은 화요일입니다."
    문자열 치환.
  • "ha".repeat(3) → "hahaha"
    문자열을 3번 반복.
  • "Hello World".indexOf("World") → 6
    "World"가 시작되는 위치.
  • parseInt("123.45px") → 123
    정수 부분만 추출.
  • parseFloat("123.45px") → 123.45
    소수까지 추출.
  • Number("123") 또는 +"123" → 123
    문자열을 숫자로 변환하는 대표적인 방법 두 가지.
  • "apple" < "banana" → true
    'a'(97)이 'b'(98)보다 코드값이 작음.
  • "Zoo" < "apple" → true
    'Z'(90)이 'a'(97)보다 코드값이 작음.
  • "가" < "나" → true
    유니코드 순서: "가"(44032) < "나"(45208).

서술형 문제 정답

  • 문자열이 기본형인데도 메서드를 쓸 수 있는 이유:
    자바스크립트 엔진이 문자열을 사용할 때 자동으로 String 객체(래퍼 클래스)로 변환해주기 때문이다.
    메서드 실행 후 다시 기본형으로 돌아간다.
  • 문자열의 불변성:
    문자열은 수정할 수 없고, 변경하려 하면 새로운 문자열이 생성된다.
    예: let s="hi"; s[0]="H"; // 여전히 "hi"
  • 문자열 비교는 코드값 기준:
    "apple" < "banana"가 true인 이유는 'a'(97)와 'b'(98)의 유니코드 값 비교 때문이다.
    즉, 사전식처럼 보이지만 실제는 숫자 값 비교.
  • 문자열 → 숫자 변환 방법:
    Number("123") → 전체를 숫자로, 변환 불가 시 NaN.
    parseInt("123px") → 정수만 추출.
    parseFloat("12.34px") → 실수 추출.
    +"123" → 단항 연산자로 변환.
  • HTML에서 특수문자를 엔티티로 표현하는 이유:
    <, >, & 같은 문자를 그대로 쓰면 태그나 구문으로 해석되기 때문에 안전하게 출력하려면 &lt;, &gt;, &amp; 와 같은 엔티티를 사용해야 한다.
