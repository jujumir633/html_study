Document와 Dom 탐색

DOM 탐색 개요

  • DOM(Document Object Model)은 HTML 문서를 트리 구조(노드들의 계층)로 표현한 프로그래밍 인터페이스
  • 탐색(Traversal)은 “어떤 노드에서 출발해 원하는 노드로 이동/검색”하는 행위

window와 document

  • window: 브라우저 탭 전체를 대표하는 최상위 객체, scrip를 실행하는 전역
  • window.document, document: 현재 로드된 HTML 문서를 나타내는 객체 (html을 렌더트리로 생성후 브라우저에 출력하는 객체), DOM 탐색의 시작점

DOM 탐색 함수 정리

  • getElementById(id): 단일 요소 반환
  • getElementsBy*([class,name,tage]) : 클래스명, 태그이름, name 명으로 요소를 찾음. 복수의 배열(HTMLCollection) 요소 반환
    • getElementsByClassName(class) : 클래스 이름으로 복수의 요소 찾기
    • getElementsByTagName(tag) : 요소 이름으로 복수의 요소 찾기
    • getElementsByName(name) : input 요소의 name 명으로 복수의 요소 찾기
    • 반환형: HTMLCollection (라이브 컬렉션)
  • querySelector(css), querySelectorAll(css) : css 선택자( >, #,. ,[])로 요소 찾기
    • querySelector: 선택자로 찾은 첫 번째 요소 반환
    • querySelectorAll: 선택자로 찾은 모든 요소를 배열(NodeList)로 반환
  • 모든 선택자는 document 뿐 아니라 특정 요소에서도 호출 가능 
    const box = document.getElementById('box');
const btn = box.querySelector('.btn');

예시 HTML

<div id="box">
  <button class="btn">버튼 1</button>
  <button>버튼 2</button>
  <input type="text" name="user" value="철수">
  <input type="text" name="user" value="영희">
</div>

탐색 코드

// 단일 요소
const box = document.getElementById("box");
console.log(box);
// <div id="box">...</div>

// 클래스명으로 찾기 (라이브 HTMLCollection)
const buttons1 = document.getElementsByClassName("btn");
console.log(buttons1);
// HTMLCollection[ <button class="btn">버튼 1</button> ]

// 태그명으로 찾기
const buttons2 = document.getElementsByTagName("button");
console.log(buttons2);
// HTMLCollection[ <button class="btn">버튼 1</button>, <button >버튼 2</button> ]

// name으로 찾기
const users = document.getElementsByName("user");
console.log(users);
// NodeList [ <input name="user" value="철수">, <input name="user" value="영희"> ]

// querySelector: 첫 번째 버튼
const btnOne = document.querySelector(".btn");
console.log(btnOne);
// <button class="btn">버튼 1</button>

// querySelectorAll: 모든 버튼 (NodeList, 정적)
const btnAll = document.querySelectorAll(".btn");
console.log(btnAll);
// NodeList(2) [ <button>버튼 1</button> ]

// 특정 요소 내부에서 탐색
const btnInsideBox = box.querySelector(".btn");
console.log(btnInsideBox);
// <button class="btn">버튼 1</button>

트리 기반 탐색(관계 이동)

  • 개념
    • DOM은 문서가 파싱될 때 이미 부모·자식·형제 관계가 연결된 트리 구조로 만들어집니다.
    • 트리 탐색은 이 관계를 따라 이동하는 방식이므로 빠르고 명확합니다.
    • 특정 요소를 기준으로 주변을 찾을 때 유리하며, 전역 검색보다 효율적입니다.
  • 자식 탐색
    • firstElementChild : 부모 요소의 첫 번째 자식 요소를 반환
    • lastElementChild : 부모 요소의 마지막 자식 요소를 반환
    • children : 모든 자식 요소들을 HTMLCollection으로 반환
  • 예시 HTML
    • <div id="parent">
  <p id="first">첫째</p>
  <p id="second">둘째</p>
  <p id="third">셋째</p>
</div>
  • 예시 실행 
    const parent = document.querySelector("#parent");

console.log(parent.firstElementChild);
// <p id="first">첫째</p>

console.log(parent.lastElementChild);
// <p id="third">셋째</p>

console.log(parent.children);
// [ <p id="first">, <p id="second">, <p id="third"> ]
  • 형제 탐색
    • nextElementSibling : 다음 형제 요소
    • previousElementSibling : 이전 형제 요소
    const second = parent.children[1];

console.log(second.previousElementSibling);
// <p id="first">첫째</p>

console.log(second.nextElementSibling);
// <p id="third">셋째</p>
  • 부모 탐색
    • parentElement : 부모 요소 반환
    console.log(second.parentElement);
// <div id="parent">...</div>
  • 조상·근접 탐색
    • closest("선택자") : 자신을 포함해 위쪽 조상 중 가장 가까운 요소
    • matches("선택자") : 요소가 선택자와 일치하면 true
    • contains(노드) : 특정 노드를 포함하면 true
    console.log(second.closest("div"));
// <div id="parent">...</div>

console.log(second.matches("p"));
// true

console.log(parent.contains(second));
// true
  • 텍스트 노드 주의
    • HTML의 공백/개행도 텍스트 노드로 인식됩니다.
    • 요소만 찾으려면 firstElementChild, nextElementSibling 등을 사용해야 합니다.

탐색 후 반환되는 타입과 특징

  • Element / HTMLElement
    • 단일 요소(태그) 노드. 대부분의 실제 조작 대상
    • 주요 메서드(요소 전용): matches(sel), closest(sel), contains(node)
    • 속성/제어(요약): classList, id, dataset, getAttribute/setAttribute, removeAttribute, style
    • 하위 범위 재탐색용: querySelector, querySelectorAll, getElementsByTagName, getElementsByClassName
    • 주의: getElementsByName은 Document/HTMLFormElement에서만 사용 가능
  • NodeList
    • 여러 노드의 “정적 또는 라이브” 모음
    • querySelectorAll → 정적 NodeList (DOM 변화 자동 반영 안 됨)
    • node.childNodes → 라이브 NodeList (변화가 즉시 반영)
    • 사용 메서드: forEach, item(i) (또는 인덱스 접근)
    • 배열 메서드 필요 시: Array.from(nodeList) 또는 [...nodeList]
  • HTMLCollection
    • 여러 요소의 “라이브” 모음(getElementsBy* 계열 반환)
    • 사용 메서드: item(i), namedItem(name)
    • 배열 메서드 직접 불가 → Array.from(collection)으로 변환 후 map/filter 등 사용
  • Node
    • 텍스트/주석/요소 등을 포함하는 상위 타입
    • 관계 접근에 공통 사용: parentNode, childNodes, firstChild, lastChild, nextSibling, previousSibling
    • 요소 전용 기능(classList 등)은 없음 → 요소 조작이 필요하면 Element인지 확인

속성/콘텐츠 접근(탐색 후 조작)

요소 객체(Element)의 속성

  • 타입: HTMLElement (Element를 상속받아 브라우저 기능 포함)
  • id : 요소의 고유 식별자 예: el.id = "title";
  • className : 요소의 클래스 이름을 문자열로 접근 예: el.className = "box highlight";
  • classList : 클래스 토글/추가/삭제를 위한 전용 객체 예: el.classList.add("active");, el.classList.toggle("on");
  • innerHTML : 요소 내부의 HTML을 문자열로 가져오거나 설정 예: el.innerHTML = "<strong>굵게</strong>";
  • textContent : 요소 내부의 모든 텍스트를 가져오거나 설정 (HTML 태그 무시, 숨김 텍스트 포함) 예: el.textContent = "안녕하세요";
  • innerText : 화면에 표시되는 텍스트만 가져오거나 설정 (CSS 스타일 반영, 숨김 텍스트 제외) 예: el.innerText = "안녕하세요";
  • style : 인라인 스타일 속성에 접근 예: el.style.color = "red";, el.style.fontSize = "20px";
  • dataset : data-* 커스텀 속성에 접근 예: <div data-user="철수"></div> → el.dataset.user === "철수"
  • on* : 이벤트 핸들러 속성. 함수(콜백)를 지정하면 해당 이벤트에 반응 예: el.onclick = (e) => { alert("클릭됨"); }; - 주의: 한 번 지정하면 기존 핸들러를 교체함 - 여러 개 이벤트를 동시에 달고 싶다면 addEventListener 사용

속성 제어 메서드

  • getAttribute(name): 속성 값 읽기
  • setAttribute(name, value): 속성 추가/변경
  • hasAttribute(name): 속성 존재 여부 확인
  • removeAttribute(name): 속성 제거
  • addEventListener("eventName", 함수): 해당 이벤트에 콜백함수 추가

클래스 제어 classList

  • el.classList.add("이름") : 클래스 추가
  • el.classList.remove("이름") : 클래스 제거
  • el.classList.toggle("이름") : 있으면 제거, 없으면 추가 (토글)
  • el.classList.contains("이름") : 해당 클래스 보유 여부 확인 (true/false)

  • 예시 HTML

    <div id="box" class="item">박스</div>

    예시 코드

    const box = document.querySelector("#box");

box.classList.add("active");
console.log(box.className);
// "item active"

box.classList.remove("item");
console.log(box.className);
// "active"

box.classList.toggle("hidden");
console.log(box.className);
// "active hidden" (처음엔 추가됨)
box.classList.toggle("hidden");
console.log(box.className);
// "active"

console.log(box.classList.contains("active"));
// true
