라이브러리와 Maven

1. 라이브러리(Library)란?

  • 다른 개발자가 미리 작성해 둔 재사용 가능한 코드 묶음.
  • 자바에서는 보통 JAR 파일 형태로 제공된다.
  • 예:
    • Oracle JDBC 드라이버(ojdbc)
    • JSON 처리 라이브러리(jackson)
    • 로깅 라이브러리(slf4j, logback)

2. 라이브러리 수동 다운로드 & 설정

  • 원시적인 방식: JAR 파일을 직접 받아서 프로젝트에 포함.
  • 절차:
    1. Oracle 공식 사이트에서 ojdbc8.jar 다운로드
    2. 프로젝트 libs/ 폴더에 저장
    3. IDE(Project Structure → Libraries)에 JAR 추가
    4. 명령줄 실행 시 classpath 지정 필요
  • 예시:
    • Windows: java -cp ".;libs/ojdbc8.jar" Main
    • macOS/Linux: java -cp ".:libs/ojdbc8.jar" Main
  • 단점:
    • 버전 충돌 해결 어려움
    • 프로젝트마다 JAR을 따로 관리해야 함
    • 자동화/협업 환경에서 불편

3. Maven 의존성 관리

  • Maven: 자바 프로젝트 빌드 및 의존성 관리 도구.
  • pom.xml 파일에 의존성을 선언하면,
    • Maven이 중앙 저장소(Maven Central)에서 다운로드
    • 로컬 저장소(.m2/repository)에 캐싱
    • 프로젝트 classpath에 자동 반영
<dependencies>
  <dependency>
    <groupId>com.oracle.database.jdbc</groupId>
    <artifactId>ojdbc8</artifactId>
    <version>23.3.0.23.09</version>
  </dependency>
</dependencies>
  • groupId: 제공 그룹(회사/조직)
  • artifactId: 라이브러리 이름
  • version: 버전 번호

4. Maven 수명주기(Lifecycle)

  • clean: 이전 빌드 산출물 삭제
  • compile: 소스 코드 컴파일
  • test: 단위 테스트 실행
  • package: JAR/WAR 산출물 생성
  • install: 로컬 저장소(.m2)에 설치
  • deploy: 원격 저장소에 배포

→ 표준화된 빌드 단계를 제공하기 때문에, 어떤 프로젝트든 동일한 방식으로 빌드 가능하다.

5. Maven과 CI/CD

  • Maven은 빌드와 의존성 관리가 자동화되어 있어 CI/CD 도구와 쉽게 연계된다.
  • Jenkins, GitHub Actions, GitLab CI 등과 연결하면
    • 코드 푸시 → 자동 빌드 → 자동 테스트 → 자동 배포
  • 수업에서는 CI/CD 자체를 다루지 않고, “Maven은 자동화 파이프라인에서 잘 쓰인다” 정도만 설명하면 충분하다.
