과제: id 없이 순서 기반 CSS 슬라이더 만들기

  • 목표 HTML과 CSS만 사용해서 이미지 슬라이더를 구현한다. JavaScript는 사용하지 않으며, id나 class 이름에 의존하지 않고 순서(nth-of-type)와 :has() 선택자만으로 동작하도록 만든다.
  • 구현 방식
    • 도트 네비게이션은 <input type="radio">를 사용해 만든다. 같은 name을 지정해 한 번에 하나만 선택된다.
    • .slider:has(input:nth-of-type(n):checked) .slides 규칙을 활용해, n번째 라디오 버튼이 선택되면 .slides 트랙을 transform: translateX()로 이동시킨다.
    • 트랙(.slides)은 display: flex;로 옆으로 나열하고, 전체 너비는 슬라이드 개수 × 100%가 된다.
    • 각 슬라이드(.slide)는 flex: 0 0 100%;를 적용해 뷰포트(.slider)의 100%를 차지한다. 따라서 translateX(-100%) 하면 정확히 한 장씩 이동한다.
    • 부드러운 전환을 위해 transition 속성을 사용한다.
  • 디자인 요구
    • 슬라이더에는 aspect-ratio: 16/9를 적용해 일정한 비율을 유지한다.
    • 도트는 hover 시 크기가 커지고, 선택되면 색상이 흰색으로 바뀌어 현재 슬라이드를 표시한다.
    • 슬라이드 안에는 텍스트 캡션, 이미지, 또는 단순한 배경 색상을 넣을 수 있다.
  • 평가 기준
    • 라디오 버튼의 순서만으로 슬라이드 전환이 정확히 동작하는가?
    • 전환 애니메이션이 자연스럽게 보이는가?
    • 슬라이더 비율과 도트 UI가 반응형 환경에서도 유지되는가?
    • 코드가 단순하고 재사용하기 좋은 구조인가?
